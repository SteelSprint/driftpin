drift — spec-drift detection for LLM coding agents

First time? Run `drift skill` for the full guide.

Usage:
  drift <command> [args]

Commands:
  init                          Initialize: create .drift/ + starter main.drift.xml
  todo                          Scan specs and markers, report drift
  list [--verbose]              Show all specs, markers, edges, and sync state
  show <marker|spec>            Show current content of a spec or marker
  diff <marker|spec>            Show what changed (unified diff vs baseline)
  diff <marker> <module.spec>   Show what changed for a specific edge
  diff --all                    Show diffs for ALL drifted edges at once (review before reset)
  link <marker> <module.spec>   Connect a marker to a spec (link-style edge)
  unlink <marker> <module.spec>  Remove a link-style edge
  reset <marker> <module.spec>  Mark a drifted link-style edge as resolved
  reset <spec> <spec>           Mark a drifted spec-spec edge as resolved (both args have dots)
  reset <id>                    Remove an orphaned (deleted) spec/marker from state.xml
  config theme <name>           Set theme preference (default, gruvbox, nord, dracula, etc.)
  config theme                  Show current theme preference
  help                          Show this help
  skill                         Print comprehensive guide (for LLM agents)
  version                       Show version

Global flags (accepted by every command):
  --json                        Output structured JSON instead of text
  --no-color                    Disable color output
  --color=auto|always|never     Control color mode (default: auto)
  --help, -h                    Show command-specific help

Exit codes (all commands):
  0                             Success / clean (drift todo: all linked + in sync)
  1                             Drift found / unlinked markers / todo pending
  2                             Error (bad args, corrupt state, I/O failure)

Themes:
  12 built-in: default, minimal, monochrome, high-contrast, dark, light,
    protanopia, solarized-dark, solarized-light, gruvbox, nord, dracula
  drift config theme <name>     Set your theme (stored in .drift/user-settings.xml)
  .drift/theme.xml              Project-level custom theme (all 18 elements, committed)

Marker syntax:
  // D! id=<markerid> range-start    Place above the code that implements a spec
  // D! id=<markerid> range-end      Place below the code that implements a spec

Spec files:
  *.drift.xml with <main> or <module name="..."> root
  Specs are <spec id="...">description</spec> — direct children of root
  Spec IDs are module-qualified: <module>.<specId>
  Refs: <ref spec="module.localid">label</ref> inside spec content
        declares a spec-spec edge; drift propagates rhizomatically

Reset dispatch (dots are the discriminator):
  drift reset <marker> <spec>   no dot, dot — link-style edge reset
  drift reset <spec> <spec>     dot, dot     — spec-spec edge reset
  drift reset <id>              single arg    — orphan removal (dot=spec, no dot=marker)

Quick start:
  drift init
  drift skill

Examples:
  drift init
  drift link validate_input core.validate_input
  drift list
  drift todo
  drift todo --json               # JSON output for LLM consumption
  drift config theme gruvbox       # set theme preference
  drift unlink validate_input core.validate_input
  drift reset validate_input core.validate_input   # link-style reset
  drift reset output.color_mode output_impl.color_presenter  # spec-spec reset
