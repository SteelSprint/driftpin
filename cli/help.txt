drift — spec-drift detection for LLM coding agents

First time? Run `drift skill` for the full guide.

Usage:
  drift <command> [args]

Commands:
  init                          Initialize: create drift.pin + starter main.pin.xml
  todo                          Scan specs and markers, report drift
  list                          Show all specs, markers, links, and sync state
  link <marker> <module.spec>   Connect a marker to a spec
  unlink <marker> <module.spec>  Remove a link between a marker and a spec
  reset <marker> <module.spec>  Mark a drifted edge as resolved
  help                          Show this help
  skill                         Print comprehensive guide (for LLM agents)

Marker syntax:
  // D! id=<markerid>          Place above the code that implements a spec

Spec files:
  *.pin.xml with <main> or <module name="..."> root
  Specs are <spec id="...">description</spec> — direct children of root
  Spec IDs are module-qualified: <module>.<specId>

Quick start:
  drift init
  drift skill

Examples:
  drift init
  drift link validate_input core.validate_input
  drift list
  drift todo
  drift unlink validate_input core.validate_input
  drift reset validate_input core.validate_input
